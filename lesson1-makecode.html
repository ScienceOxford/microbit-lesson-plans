Lesson 1: introduction to the BBC micro:bit


Step 1: What is a BBC micro:bit?

A micro:bit is a small microcontroller with built-in features such as:

  • display
  • buttons
  • motion sensors
  • temperature sensor
  • wireless transmission
  • connection pins

All of these features are programmable, and you will introduce them all to your students though these lessons.

More details and a diagram can be seen on microbit.org.

During lesson 1, you will be using the display and the buttons.


Step 2: The MakeCode Editor

To write code for the micro:bit, open the MakeCode editor by visiting https://makecode.microbit.org, then clicking New Project.

MakeCode runs in a web browser, and needs an internet connection the first time it is accessed. It will then be cached in your browser so, if necessary, internet access can be revokes and it will still run.

Alternatively, you can download apps for:

  • Windows 10
  • Android mobile / tablets
  • iOS mobile / tablets

The apps work similarly, but for the rest of these instructions it is assumed that you are using MakeCode in a browser.

  1. Click the Home button to access tutorials, and to start a new project.
  2. Click here to switch between Blocks programming (which will be used throughout these lessons), and Javascript programming.
  3. The Settings button will be used to pair micro:bits with the computer (see later instructions).
  4. The Simulator allows you to test your code before downloading it to a physical micro:bit.
    On smaller screens this may not be visible by default, so click the grey arrow on the right to open and close this section.
  5. Drag and drop blocks from the Menu to create your programs.
  6. Drop blocks into this section to form your programs. To delete blocks, drag and drop them onto the Menu.
  7. Clicking the Download button will allow you to run code on a physical micro:bit (see later instructions).
  8. Use this box to Rename your file.

Step 3: Displaying Pictures & Messages

When you create a new MakeCode project, two blocks will already be on the screen for you to use:

  • Inside the on start loop is where you will place blocks you want to happen at the beginning of the program (similar to 'when the green flag is clicked' in Scratch).
  • The forever loop is where you will place blocks that you want to repeat until the program is restarted.

These two blocks do not do anything on their own - you must drag and drop blocks of code inside them.

The blocks to show pictures and messages on the micro:bit's display can be found in the Basic menu.

  • Use the show icon block to choose from a collection of pre-made images.
  • The show leds block lets you create images by clicking the squares to turn on specific LEDs.
  • To scroll messages, use the show string block. Click inside the speech marks to change the text.

A string is a data type in programming languages that holds information (including words and numbers) inside speech marks or apostrophes.

This is in comparison to an integer (a whole number), which can only be a number and therefore is easier to do maths with.

Both data types are useful for different things, and we will use both in future lessons.

Introduce your students to these blocks, then give them time to tinker and explore the options within the Basic menu. Their code will run in the Simulator.

Encourage your students to ask each other for help debugging their code if it does not run as expected, and to share ideas with each other.

Extension idea

By combining pictures with pause blocks, you can create animations.

Change the number in this block to add longer or shorter pauses.

Try creating a set of images to make a character move across the screen.

Does it make a difference if your animation is inside on start or forever?

Pairing your BBC micro:bit & Downloading Code

The way this works will depend on the device you are programming the micro:bit from, and your settings – it is highly recommended that you test this out before running the activity with your students, using their devices.

The below instructions assume you are using a browser version of the MakeCode editor.

Until now, your students have been using the Simulator. Make sure that they have a working program before downloading it to the micro:bit!

Connect the micro:bit to the computer using the USB cable (it may take a moment to install drivers). It will be visible to your computer as though it were a USB stick or similar, and will be named MICROBIT.

Follow these instructions to pair your micro:bit.

If successful, clicking the Download button will send your code directly to the micro:bit.

If unsucessful, you can put code onto the micro:bit by dragging and dropping it from it's download location on your computer, onto the MICROBIT drive.

Check that the code running on the micro:bit is the same as that running in the Simulator!


Step 5: Using the A & B Buttons

So far, your students have been outputting information to the micro:bit. You will now introduce them to inputs.

Ask them if they can name any outputs and inputs they use in regular life.

Example outputs: computer monitor, phone/ ipad screen, text message received…

Example inputs: computer mouse, light switch, text message typed…

In this lesson, your students will use the A & B buttons on the front of the micro:bit as inputs.

The blocks to control inputs are in the Input menu.

The on button A pressed block can be used instead of on start or forever to program when other blocks will run.

By clicking the arrow next to A in this block, it can be modified to use button A, B or A+B as the input.

Challenge your students to make one thing happen when button A is pressed, and something else happen when button B is pressed.

Encourage your students to keep downloading their updated programs to the micro:bit (instead of using the Simulator), to get used to the process.
